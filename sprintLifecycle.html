Tracking User Stories through Sprint Cycle

Purpose

This tutorial walks through using Trello during your Sprint Cycle.

What is a Sprint Cycle

A sprint cycle is an incremental two to four week period where a Scrum team works toward a potentially shippable product. The teams works through all the tasks required to meet the user story's acceptance criteria. When all tasks are completed it is reviewed by the team and tested before accepting as done.

Step-By-Step Instructions

Pulling Tasks

    Reminder: Pulling a task means each team member chooses a task from the Sprint Backlog to complete.
  1. Review Current Sprint Backlog and the tasks for each user story
  2. Choose one to two tasks to start working on from the Sprint's user stories

    3. Note: Do not put a checkmark next to the task. This is done when the task is complete. We are just looking at what we want to work on.

  3. Once you have chosen a task(s), make sure the user story is selected
  4. On the right-hand side under the sub memnu Add, select Members
  5. Select your username

    6. Note: This is used to show others that you are working on a task in the user story.

  6. Press the x to the right of Members pop-up to close out the box
  7. You should now see your icon under the user story

Documenting what you are working on and moving a user story to In Progress

    Note: Since tasks can be pulled from any team member, it is helpful to document what task you are working on.
  1. While in the user story scroll to the bottom under Add Comment
  2. Type Working on and type the task(s) you want to work on.
  3. Click Send to save
  4. Next move the user story working on from Current Spring Backlog list to In Progress list

    5. Note: Now other team members know what tasks are remaining to finish a user story.

Completing a Task

    Note: When a task is complete make sure to check documentation and push the latest version of your code to repository.
  1. Click on the user story that contains the completed task
  2. Put a checkmark for the task completed

    3. Note: The Activity section of user story documents which team member completed the task

  3. Check if additional tasks needed to complete and repeat Pulling Tasks

All Tasks Completed on a User Story

    Note: Once all tasks are complete the team will review documentation, code, and tests.
  1. Create a new checklist called Confirmed Review
  2. Click Add Item for Confirmed Review
  3. Add all of your team member names

    4. Note: This is so all team members are accountable in reviewing the tasks and code

  4. Move user story from In Progress to Review

    5. Note: The review process is made up of testing implemented code, reviewing definition of done, and make sure acceptance criteria met for the user story.

  5. Go through your review process to confirm Acceptance Criteria and Definition of Done met
  6. Open user story once review completed
  7. Put a checkmark next to the criteria met.

    8. Note: If any Acceptance Criteria is not met then write a comment why, add additional required tasks and move card back to Current Sprint Backlog

  8. Once all Acceptance Criteria met, then each member puts a checkmark next to their name
  9. Move user story from Review to Done

This is one of many ways to use Trello. I encourage everyone to try what works best for them. Make sure to review with your teacher on any requirements that need to be met.
